Geometry Chaos Editor Help

Object editor

Object editor allows you to edit every property of an object. It also features a keyframe editor, using which you can move and animate your objects.

Objects have fixed and animated properties. Fixed properties are the ones that are set once and cannot change during the playthrough of a level. Animated are the ones that can be changed at any time using keyframes.

Fixed properties

  • Name: a name of an object, may not be unique. If objects have the same name, they are considered as parts of an array. Also, with the special syntax you can set an object as a child of a parent object.

  • Start time: time at which object is appearing (in milliseconds).

  • Duration: how long the object should be on screen (in milliseconds).

  • View order: controls the layering of objects. Objects with lower view order will be in front of objects with higher view order.

  • Collidability: controls whether an object should hit players. Always = hit, Never = don't hit. Opacity-based means based on Highlight property of an object: when it is negative, object won't hit players, otherwise it will.

  • Shape: lets you change an appearance of an object. Some shapes can have unique settings.

Shapes settings

Text

  • Text: the text to display.

  • Origin: Where the origin point should be located relative to the text. You can also think of it as a text alignment.

Animated properties

  • PosX: x position of an object on the game field (in pixels). X-axis is pointed from the left to the right, 0 is at the center.

  • PosY: y position of an object on the game field (in pixels). Y-axis is pointed from the top to the bottom, 0 is at the center.

  • ScaleX: scaling of an object along the x coordinate (multiplier).

  • ScaleY: scaling of an object along the y coordinate (multiplier).

  • Rot: rotation of an object (in degrees). Notice that rotation affects direction of scaling (axes are rotated with the object).

  • PivotX: x position of pivot point relative to the origin of the object (usually center) (in pixels). More on pivot point later.

  • PivotY: y position of pivot point relative to the origin of the object (usually center) (in pixels).

  • Highlight: controls opacity and highlight of an object. Values from -1 to 0 change object's opacity from completely transparent to completely opaque. Values from 0 to 1 change object's brightness from normal to completely bright (white).

  • Stroke: controls stroke width of an object. If value is ≤ 0, then the object is filled. Otherwise, object is stroked and the value sets the stroke width.

Pivot point

Pivot point is the point around which scaling and rotation transformations are made. By default, the position of a pivot point is set to (0, 0), which means that the object scales and rotates around its origin (center).

Let's consider the following example. Imagine you have an object in the shape of a square. If its pivot is set to (50, 50), it will be located on the bottom-right angle of the square (regardless of object's position or scale). This means that object will be rotated and scaled around bottom-right angle of the square.

Keyframe editor

Keyframe editor is located right after the fixed properties. It consists of two parts: keyframe tracks and keyframe properties editor.

Tracks show every keyframe for every animated property. By default, all objects already have one keyframe for each property, which are located at the very beginning of tracks (keyframes are visualised as circles). They are setting default values for properties. You can click on keyframes to see their properties (more on them later).

The point of keyframes is to enable animations for object. Game will interpolate values between two consecutive keyframes, so object will appear to be moving. If track has only one keyframe, object will not be changing and will always have the value of this keyframe. When last keyframe of track is reached, object will not be changing and will have the value of this last keyframe.

You can notice that there is a cursor on the tracks, which moves together with the cursor from timeline. This cursor is here to show you which keyframes are being active and interpolated at the moment (that is the nearest keyframe from the left and nearest from the right side of cursor).

To create new keyframes click on the plus sign on the left of the track, that you want to add keyframe to. It will be created at the position of your cursor. If there is already a keyframe at this position, editor will instead select this keyframe. Alternatively, you can click at the keyframe itself to select it. To delete a keyframe, select it and press Delete. Selecting a keyframe will show you its properties.

Keyframe properties

  • Time: time at which the keyframe is located relative to the start time of the object (in milliseconds). The number to the right of text field is the time of this keyframe relative to the beginning of the level (basically keyframe time + object start time).

  • Value: the value of this keyframe which is the end value of animated property (i.e. what value should the animated property have when this keyframe is reached).

  • Interpolator: a function which defines how values should be interpolated.

  • Randomizer: randomizes value for each play of the level.

Interpolates

Each interpolator has its own function to determine the value of property at the exact time. For example, with the 'Linear' interpolator and keyframes set at timings of 0 and 1000 with values of 10 and 50 respectively, at time 500 (halfway between 0 and 1000) the value of animated property will be equal to 30 (halfway between 10 and 50). Try each of the interpolates to find out how they are affecting your animation!

Randomizers

If you want to randomize your level a bit, you can use randomizers for keyframes. They will set a random value for keyframe for every launch of your level. There are two types of randomizers:

  • Linear: chooses a random value between min and max values (1st and 2nd fields respectively).

  • Discrete: chooses a random value between min and max values with specified step (3rd field). For example, with min and max values set to -20 and 10 and step set to 5 randomizer will choose between the following values: (-20, -15, -10, -5, 0, 5, 10).
