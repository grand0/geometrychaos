Parenting

Parenting is convenient when you want to create something like a custom object. Parenting means that there can be objects-children which inherit some properties from object-parent. In other words you can control properties of multiple objects using another object.

How to use

Any object can act as a parent. To create a child, you need to name the object-child in a certain way: {parent name}/{child name}. {parent name} must be exact match to the name of object-parent. {child name} can be anything you like. Child should immediately inherit some of the properties of the parent.

Parent must be on screen for inheriting to work. When parent disappears parenting stops working immediately.

Inheriting

Not all properties of the object are inherited, and ones that are inherited do that in different ways.

Fixed properties

Only View order gets inherited. Parent's view order gets added to the child's view order.

Animated properties

All animated properties are getting inherited except the Stroke property. All parent's animated properties are getting added to the child's ones, except for the ScaleX and ScaleY, which are getting multiplied.
